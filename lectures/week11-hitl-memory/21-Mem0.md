# Mem0 and Dedicated Memory Layers

Managed memory-as-a-service for agent systems.

## The case for a memory layer
- Building memory from scratch is hard: extraction, deduplication, retrieval, decay
- Dedicated memory services handle the complexity
- Focus on your agent logic, not memory infrastructure

## Mem0: memory-as-a-service

```python
from mem0 import Memory

# Initialize with configuration
config = {
    "llm": {"provider": "openai", "config": {"model": "gpt-4o-mini"}},
    "vector_store": {"provider": "chroma", "config": {"collection_name": "memories"}},
}
memory = Memory.from_config(config)

# Add memories from conversation
memory.add(
    "I prefer dark mode and use VS Code as my editor",
    user_id="alice",
    metadata={"source": "preferences"}
)

# Memories are automatically extracted, deduplicated, and stored
# "I prefer dark mode" and "I use VS Code" become separate memory entries

# Search relevant memories
results = memory.search("What editor does Alice use?", user_id="alice")
# Returns: [{"memory": "Uses VS Code as editor", "score": 0.95}]

# Get all memories for a user
all_memories = memory.get_all(user_id="alice")

# Update conflicting memories automatically
memory.add("I switched to Cursor from VS Code", user_id="alice")
# Old "uses VS Code" memory is updated to "uses Cursor"
```

## What Mem0 handles automatically:
- **Extraction** — Identifies memorable facts from conversations
- **Deduplication** — Avoids storing the same fact twice
- **Conflict resolution** — Updates stale memories when new info contradicts
- **Retrieval** — Semantic search with relevance scoring

## Alternatives and similar tools:
- **Zep** — Long-term memory with temporal awareness and entity graphs
- **Letta (MemGPT)** — Self-editing memory with tiered storage
- **Custom solutions** — Vector DB + extraction prompts + cron cleanup

## When to use managed vs. custom:
- **Managed (Mem0/Zep):** Fast to integrate, good defaults, limited customization
- **Custom:** Full control, specific domain needs, willing to maintain infrastructure
