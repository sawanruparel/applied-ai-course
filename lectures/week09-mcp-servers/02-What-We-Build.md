# What We're Building Today

A **custom MCP server** that bridges an LLM to a real enterprise system.

By the end of this lab you will have:

1. A working MCP server with tool definitions and handlers
2. Proper input validation using JSON Schema
3. Error handling and safety guardrails
4. A server you can connect to Claude Desktop or any MCP host

**Choose your track:**

- **SQL Track** — Query a SQLite/PostgreSQL database through natural language
- **Jira Track** — Search, create, and update Jira issues via the API
- **Slack Track** — Search messages, post to channels, list channels

All tracks follow the same MCP server patterns. The only difference is what your tool handlers call.
