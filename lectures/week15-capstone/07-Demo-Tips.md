# Demo Tips: Making It Real

### Keep It Authentic
- Use real (or realistic) data — not "lorem ipsum" placeholders
- Show the actual system running, not a screen recording of a perfect run
- Let the audience see the agent's reasoning traces and tool calls

### Show Failure Handling
- Deliberately trigger an edge case — how does the system recover?
- Demonstrate guardrails in action (Week 14: safety patterns)
- Show what happens when a tool call fails or returns unexpected data

### Have a Backup Plan
- Record a backup video of the demo in case of live failures
- Have screenshots of key outputs ready
- Know your system's weak points so you can narrate around them

### Demo Flow
1. State the scenario in plain language
2. Show the input (user query, trigger event)
3. Narrate the agent's reasoning as it executes
4. Highlight the MCP tool calls and their results
5. Show the final output and explain why it's good

---

**The best demos tell a story. Don't just show features — show a problem being solved.**
